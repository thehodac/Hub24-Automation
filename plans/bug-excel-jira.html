Plan — Export bugs to Excel/CSV in the Rovo/Jira format

Author: Thong Ho · Date: 2026-07-15 · Template: NIT-6180.xlsx (Jira export) · Related: reporter bug-report

1. Goal

After each test run, the reporter automatically exports all bugs into one .xlsx file + one .csv file in the exact column format of the template file NIT-6180.xlsx, saved into the bug-excel-jira/ folder, so that Rovo/Jira can import them to create bug tickets. File name = the Epic key in .env (JIRA_PARENT_KEY, e.g. NIT-6180.xlsx).

2. Scope

  • In scope: add a table-export function to reporters/bug-report.ts; create the bug-excel-jira/ folder; write .csv (no library needed) + .xlsx (add exceljs).
  • In scope: merge bugs of all types (e2e/api/accessibility/chromatic) into the same file, one bug per row.
  • Out of scope: auto-importing into Jira (you do it manually / Rovo reads the file). Do not touch the existing bug→Jira REST flow.

3. File format (per the NIT-6180.xlsx template)

Keep the 32-column header row of the export file; only fill the columns that have data, leave the rest blank:

ColumnValue to fill
Keyleave blank (Jira assigns it on creation)
Summary[Bug] <describe> - <title>
DescriptionJira wiki markup (see section 4)
Labels<severity>, autobug-<hash>, automation
ParentJIRA_PARENT_KEY (from .env)
ProjectJIRA_PROJECT_KEY (NITRO)
P (Priority)mapped from severity: P1→Highest · P2→High · P3→Medium · P4→Low
T (Type)Bug
Creator / Reporter(optional) a fixed name via env, or leave blank
the remaining custom columnsleave blank (Cause of Issue, Compliance required, FDS Approvers, …)

<step>

h3. Actual result (current)

<actual>

h3. Expected result (to pass)

<expected>

Source: <category> — <file>

Created automatically by the HUB24 Bug Report reporter.

5. File name

  • bug-excel-jira/<JIRA_PARENT_KEY>.xlsx and <JIRA_PARENT_KEY>.csv — e.g. NIT-6180.xlsx.
  • Each run overwrites the file with the same epic name (just like the template). If you want to keep a per-run history → add a timestamp <EPIC>-YYYYMMDD-HHmm.xlsx (let me know).

7. How to verify

  • npm install then npm run test:saucedemo (which has a failing case).
  • Check that bug-excel-jira/<EPIC>.xlsx + .csv have the correct columns as in NIT-6180.xlsx, the Description has correct wiki markup, and Parent = the epic in .env.
  • Try a Rovo/Jira import of that file to create tickets.

8. Risks / notes

  • exceljs is new; if installation is blocked by the network, .csv still works (fallback).
  • The template export has many unrelated custom fields (SFTP, FDS…) → leave them blank, do not generate data.
  • Does not affect the existing bug→Jira REST flow; this is a parallel export channel.
