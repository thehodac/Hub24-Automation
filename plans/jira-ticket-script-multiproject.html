Plan — Reusable multi-project Jira ticket script

Author: MPF Developer · Date: 2026-07-03 · Replaces the Cowork slash command (not supported)

1. Goal

A reusable Node script that creates a Jira ticket (and optional sub-task) in ANY project — not hard-coded to NITRO — so the ticket-creation step can be run standalone or called as one step inside a larger workflow.

2. Scope

  • In scope: one parameterised script + an npm script to run it; reuse existing Jira logic.
  • Out of scope: a Cowork "skill" (must be created via Settings > Capabilities, not here); the larger multi-step workflow (added later, will call this script).

3. How it works

  1. Run: node scripts/create-jira-ticket.mjs --project HUB --type Bug --summary "App Header - Login button missing" [--parent NIT-5186] [--subtask "..."]
  2. Reads Jira credentials from .env (JIRA_BASE_URL / JIRA_EMAIL / JIRA_API_TOKEN).
  3. Prints the planned fields and asks for confirmation (unless --yes is passed for automated runs).
  4. Creates the ticket via the Jira REST API; optionally creates a sub-task inside it.
  5. Prints the created key + URL.

4. Parameters

6. How to verify

  • Run with a test project/parent → a ticket is created with the right fields.
  • Run with --subtask → a sub-task appears inside it.
  • Delete the test tickets afterwards.

7. Risks / notes

  • Creates REAL tickets — confirmation is ON by default; use a clear prefix while testing and delete after.
  • Credentials stay in .env (never committed).
  • Later the big workflow script will call this as one step.
