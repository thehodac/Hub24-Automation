Plan — Slash command: /create-jira-ticket

Author: MPF Developer · Date: 2026-07-03 · Goal: reuse the Jira ticket-creation workflow on demand

1. Goal

Create a reusable slash command so that typing /create-jira-ticket (with a short description) makes Claude create a Jira ticket following the team convention — without re-explaining the steps every time.

2. Scope

  • In scope: one project command file that defines the ticket-creation workflow (type, project, Epic, naming, optional subtask).
  • Out of scope: the automated reporter (already built for test failures); creating a "skill" (not possible in this session).

3. What the command will do

  1. Take the bug/short description from the command arguments.
  2. Create a Jira issue: project NIT, type Bug (default), under Epic NIT-5006 (configurable in the command text).
  3. Name it <Func Spec> - <short description>.
  4. Optionally create a sub-task inside it if asked.
  5. Return the created ticket key + link, and remind it can be deleted after testing.

4. Files to create / change

FileChange
.claude/commands/create-jira-ticket.mdNew — the slash command prompt template

5. How to verify

  • Type /create-jira-ticket in the chat — the command appears.
  • Run it with a sample description → a ticket is created under the chosen Epic with the right name.
  • Delete the sample ticket afterwards.

6. Risks / notes

  • If Cowork does not surface project slash commands, we switch to a Skill (via Settings > Capabilities).
  • Command creates REAL tickets — keep a clear naming prefix (e.g. [QC-Test]) while testing, delete after.
  • Default Epic/project are written in the command text and can be changed anytime.
