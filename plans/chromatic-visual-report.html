Plan — Report for Chromatic: before/after/diff images + Chromatic link

Author: MPF Developer · Date: 2026-07-07 · Option 3 (combined)

1. Goal

On each visual run, if there is a UI change → create a report file containing the before (expected) image, after (actual) image, and diff image, along with a link to the Chromatic build to open/review it on the dashboard.

2. Core idea

  • Local images come from Playwright visual testing (toHaveScreenshot()): when an image differs from the baseline, the test FAILS locally and Playwright automatically generates the 3 images -expected.png / -actual.png / -diff.png.
  • Report comes from the existing reporter: for the chromatic type, embed all 3 images into the .docx file and add a Chromatic link line.
  • Chromatic link comes from the chromatic run (the CLI prints the build URL); save the URL into a file for the reporter to read, or set it via the env var CHROMATIC_BUILD_URL.

3. Steps

  1. Visual specs (tests/chromatic/): use await expect(page).toHaveScreenshot('<name>.png') to compare locally (replacing / supplementing the current archive approach). The first run creates the baseline.
  2. playwright.config: configure the visual project to allow writing the baseline (--update-snapshots on the first run) and store snapshots next to the spec.
  3. reporters/bug-report.ts: for the chromatic category, find the 3 image attachments from toHaveScreenshot (named "expected"/"actual"/"diff"), copy them into bug-image/chromatic/ and embed all 3 into the .docx ("Before / After / Diff" section). Add a Chromatic build: <link> line.
  4. Chromatic link: add a npm run test:visual:report script that runs chromatic then writes the build URL to .chromatic-build-url; the reporter reads this file (or the env var CHROMATIC_BUILD_URL) to insert the link. If it is missing → the report still has the images, only the link is absent.
  5. .gitignore: ignore temporary diff images and .chromatic-build-url.

5. How to verify

  • Run once to create the baseline. Make a small change to a component → run npm run test:visual again.
  • Expected: bug-report/chromatic/[FAIL][ISSUE-NN] ....docx has 3 before/after/diff images + the Chromatic link; the images are in bug-image/chromatic/.

6. Risks / notes

  • toHaveScreenshot needs a baseline — the first run must create the baseline (this does not count as a failure).
  • The comparison images are Playwright's (local), independent of the images on Chromatic; the link only opens the dashboard.
  • An exact deep-link to each change needs the Chromatic build number; if it cannot be obtained, use the general build link.
  • An image change may be intentional — a person still needs to confirm it (accept the new baseline) as usual.
