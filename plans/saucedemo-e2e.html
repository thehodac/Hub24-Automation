Plan — SauceDemo E2E automation (demo for Kat)

Author: Thong Ho · Date: 2026-07-10 · Source: Confluence "Saucedemo_TestCase" (36 test cases) · Related: e2e demo for Kat/Paulo

1. Goal

Automate the 36 test cases of saucedemo.com (5 modules) with Playwright + Page Object Model, runnable with a single command npm run test:saucedemo to record a demo video of the e2e test run. This suite is independent of the HUB24 environment (no HUB24 login needed, no test-env access that is currently blocked).

2. Scope

  • In scope: 36 test cases — Login (8), Inventory (10), Cart (6), Checkout (11), Navigation (3). The tests are written to pass against the site's real behavior.
  • In scope: page objects for SauceDemo, dedicated fixtures, a dedicated Playwright project, an npm script.
  • Out of scope: problem_user / error_user / visual_user (noted as a Risk in the test case page) — not done in this demo version.
  • Out of scope: deliberately creating a failing bug (use the existing _demo-fail* files when needing to demo Jira ticket creation). This suite aims for 100% pass.

3. Step-by-step approach

  1. Create page objects in pages/saucedemo/ (extend BasePage, locators in the constructor, prefer data-test).
  2. Create dedicated fixtures pages/saucedemo/fixtures.ts injecting the page objects (specs do not use new).
  3. Write 5 specs by module in tests/saucedemo/, each scenario mapping to one test case (TC_ID noted in the test title).
  4. Add the saucedemo project to playwright.config.ts (baseURL = saucedemo, not dependent on setup, no storageState) and have the default projects skip this folder.
  5. Add the test:saucedemo script to package.json.
  6. Do a trial run, adjust the selectors to match the site, ensure 100% pass.

5. How to verify

  • npm run test:saucedemo → 36 tests pass (chromium).
  • npm run test:saucedemo -- --headed to record a demo video for Kat (seeing the browser act).
  • npx playwright show-report to view the HTML report.
  • No Jira ticket created (since everything passes).

6. Risks / notes

  • Separate project, does not touch HUB24 (BASE_URL, storageState, auth.setup) — safe.
  • SauceDemo uses data-test which is very stable → durable selectors.
  • TC_LOGIN_007 (performance_glitch_user) runs slowly, up to ~10s → increase the timeout just for that test.
  • TC_CHK_007 (tax/total): assert item total + tax + total > 0 and total = subtotal + tax, instead of hardcoding the amount.
  • This is a public sample site; running it many times does not affect HUB24 data.
