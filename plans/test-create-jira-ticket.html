Plan — Test creating a Jira Bug ticket from a failed test

Author: MPF Developer · Date: 2026-07-03 · Related: bug-report reporter + reporters/jira.ts

1. Goal

Verify end-to-end that when a Playwright test fails, the automation creates a real Bug ticket in Jira project NIT, linked under Epic NIT-5006 (App Header), named <Func Spec> - <short bug>, with the screenshot and video attached — matching Paulo's requirements.

2. Scope

  • In scope: run ONE demo failing test locally, create ONE real ticket, verify it, then delete it.
  • Out of scope: enabling ticket creation on CloudBees CI; running against real App Header tests; changing reporter code (already built).

3. Step-by-step approach

  1. Create a Jira API token at id.atlassian.com → API tokens.
  2. In .env (gitignored) set: JIRA_CREATE_TICKETS=true, JIRA_EMAIL, JIRA_API_TOKEN (project=NIT, epic=NIT-5006 already set).
  3. Run the demo failing test: npx playwright test tests/UI/_demo-fail.spec.ts --project=chromium
  4. Read the console: expect Created NIT-xxxx.
  5. Open the ticket in Jira; check the fields + attachments (see section 5).
  6. Delete the demo ticket so it does not clutter the board.

4. Files to create / change

FileChange
.env (local only, gitignored)Fill JIRA_EMAIL + JIRA_API_TOKEN, set JIRA_CREATE_TICKETS=true
(no source code changes)Reporter + Jira integration are already implemented

5. How to verify

  • Console prints ✅ Created NIT-xxxx.
  • Ticket type = Bug, under Epic NIT-5006.
  • Title = Demo bug report - intentionally fails to demo the bug report.
  • Description has: Reason, Failed step, Steps to reproduce, Actual, Expected.
  • Attachments: 1 screenshot (.png) + 1 video (.webm).

6. Risks / notes

  • Creates a REAL ticket in the NITRO project — must delete the demo ticket after checking.
  • API token is a secret → only in .env (never committed).
  • If it fails: 401 = wrong email/token; 400 = field/epic-link issue (we then adjust).
  • Later: rename the demo describe to a real Func Spec (e.g. "App Header") for real runs.
