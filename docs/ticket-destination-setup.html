Configure the ticket destination (per project)

Updated: 2026-07-17 · Each MPF project uses a different tool → configure .env correctly before running.

REQUIRED workflow before enabling auto-ticket

  1. Ask which tool the project uses: Jira, Asana, Azure DevOps, or Basecamp? (May be more than 1.)
  2. Fill in .env for that tool (sections below), and set the *_CREATE_TICKETS=true flag.
  3. Run one failing test to check the ticket is created in the right place.

Only enable the tools the project actually uses. Any tool not in use → leave *_CREATE_TICKETS=false (the default). Reporters run in parallel, independently per tool. NEVER commit a token/PAT/secret — keep them only in .env (gitignored).

1. Jira

JIRA_CREATE_TICKETS=true
JIRA_BASE_URL=...   JIRA_EMAIL=...   JIRA_API_TOKEN=...
JIRA_PROJECT_KEY=...   JIRA_PARENT_KEY=...
VariableHow to get it
JIRA_BASE_URLYour Atlassian site URL, e.g. https://classltd.atlassian.net.
JIRA_EMAILYour Atlassian account email (an account with permission to create issues).
JIRA_API_TOKENid.atlassian.com/manage-profile/security/api-tokens → Create API token → copy (shown once).
JIRA_PROJECT_KEYThe project key (e.g. NIT) — visible at the start of a ticket key (NIT-123) or in Project settings.
JIRA_PARENT_KEYThe Epic/Story key to attach the ticket to (e.g. NIT-6655).

2. Asana

ASANA_CREATE_TICKETS=true
ASANA_TOKEN=...   ASANA_PROJECT_GID=...   ASANA_SECTION_GID=   (empty = default section)
VariableHow to get it
ASANA_TOKENAvatar → Settings → Apps → at the bottom of the page "View developer console" → Personal access tokens → New token → copy (shown once).
ASANA_PROJECT_GIDOpen the project → URL: .../project/1207751943301534/... → the number after /project/.
ASANA_SECTION_GID(optional) run node scripts/asana-sections.mjs <PROJECT_GID> → it prints the section names + GIDs → copy the GID you want. Empty = default section.

3. Azure DevOps (Azure Boards)

AZURE_DEVOPS_CREATE_TICKETS=true
AZURE_DEVOPS_ORG=...   AZURE_DEVOPS_PROJECT=...   AZURE_DEVOPS_PAT=...
AZURE_DEVOPS_AREA_PATH=   (optional)
VariableHow to get it
AZURE_DEVOPS_ORGThe organization name in the URL: dev.azure.com/<org>.
AZURE_DEVOPS_PROJECTThe project name (visible in the URL after the org, or in the Projects list).
AZURE_DEVOPS_PATGo to dev.azure.com → top right User settings → Personal access tokens → New Token → select the Work Items (Read & Write) scope → Create → copy (shown once).
AZURE_DEVOPS_AREA_PATH(optional) Project settings → Boards → Team configuration → Areas. Can be left empty.

4. Basecamp 3

BASECAMP_CREATE_TICKETS=true
BASECAMP_TOKEN=...   BASECAMP_ACCOUNT_ID=...   BASECAMP_PROJECT_ID=...   BASECAMP_TODOLIST_ID=...
VariableHow to get it
BASECAMP_ACCOUNT_IDThe number in the Basecamp URL: 3.basecamp.com/1234567/....
BASECAMP_TOKENBasecamp uses OAuth2: register an app at launchpad.37signals.com/integrations → run the OAuth flow to obtain an access token. (More involved than a regular token — usually get a dev/admin to help.)
BASECAMP_PROJECT_IDOpen the project → URL: .../buckets/<project_id>/... → the number after /buckets/.
BASECAMP_TODOLIST_IDOpen the exact to-do list you want to create tasks in → URL: .../todolists/<todolist_id> → the number after /todolists/.

After filling it in

npx tsc --noEmit          # check syntax
npm run test:e2e          # (or any test command) - a failing test will create a ticket in the enabled destination

Check that the ticket appears in the configured tool. If you enable multiple tools at once, tickets are created in parallel in all of them.
