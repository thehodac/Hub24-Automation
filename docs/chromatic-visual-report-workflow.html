Workflow: Chromatic Visual Report (before/after/diff images + link)

Automation QC · Playwright + Chromatic · Updated: 2026-07-07

When running visual tests, if the UI changes compared to the baseline → generate a report file (.docx) containing Before / After / Diff images (compared locally by Playwright) and a link to the Chromatic build (where changes are reviewed/approved on the dashboard).

Flow: run the visual test → Playwright toHaveScreenshot() compares with the baseline → if different: test FAIL, produce expected/actual/diff images → the reporter embeds the 3 images into bug-report/chromatic/…docx → upload to Chromatic to create a new build → get the build link → insert it into the report.

1. How to run

One command (recommended) — run the test, create the Chromatic build, and insert the link into the report:

npm run test:visual:chromatic

This command automatically does 3 steps: (1) run the visual test → report + images, (2) upload to Chromatic → save the build URL to .chromatic-build-url, (3) run the visual test again so the report has the link. Requires CHROMATIC_PROJECT_TOKEN in .env.

Or split the commands: npm run test:visual (images only, no link) · npm run chromatic:report (create build + save link).

2. First time — create the baseline

toHaveScreenshot needs a reference image. The first run writes the baseline (the test "fails" once when creating it, and passes on the next run):

npx playwright test tests/chromatic/<spec> --project=visual --update-snapshots

The baseline is stored in tests/chromatic/<spec>-snapshots/. When a change is approved (matching the new design), update the baseline with --update-snapshots.

3. Result (when there is a change)

  • bug-report/chromatic/[FAIL][ISSUE-NN] <test>.docx — has a Visual comparison section: Before (expected) / After (actual) / Diff, and the line Chromatic build: <link>.
  • Images in bug-image/chromatic/.
  • On Chromatic: a new build with comparison images + Accept/Deny buttons.

4. Related files

FileRole
tests/chromatic/*.spec.tsImport @chromatic-com/playwright (archive for Chromatic) + toHaveScreenshot() (local image comparison).
reporters/bug-report.tsChromatic type: embeds the 3 expected/actual/diff images + the Chromatic link line (read from env CHROMATIC_BUILD_URL or the file .chromatic-build-url).
scripts/visual-with-chromatic.mjsCombines into 1 command: test → chromatic → test (so the report has the link).
scripts/chromatic-build-url.mjsRuns Chromatic, captures the build URL, saves .chromatic-build-url.

5. Configuration (.env)

  • CHROMATIC_PROJECT_TOKEN — Chromatic upload token (DO NOT commit).
  • CHROMATIC_BUILD_URL — (optional) manually set the build link for the report to use instead of the file.

6. Notes

  • The comparison images are Playwright's (local); the link only opens the Chromatic dashboard.
  • The link is currently the build page (listing all changes), not yet a deep-link to a single component.
  • A UI change may be intentional → someone still needs to Accept the new baseline on Chromatic.
  • If Chromatic reports an outdated CLI: upgrade with npm i -D chromatic@latest.
